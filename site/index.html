Figma plugin · WCAG 2.2 AA · Local-first AI

Catch accessibility issues
before they ship.

Figma A11y Audit scans your design file against WCAG 2.2 AA — text contrast, tap targets, text size, focus states — and annotates tab order and alt text with a local vision model via Ollama. Nothing leaves your machine.

Six Tier-1 checks

  • 01 · WCAG 1.4.3Text contrastPer-segment ratios, alpha blending, large-text thresholds, gradient/image warnings.
  • 02 · WCAG 1.4.11Non-text contrastBorders, icons and UI elements against their effective background.
  • 03 · WCAG 2.5.8Tap target size24px AA minimum with spacing analysis against neighboring interactive targets.
  • 04 · WCAG 1.4.4Text size12px recommended minimum, 10px hard floor, caption/legal exceptions.
  • 05 · WCAG 2.4.7Focus state definedFlags interactive component sets shipping Hover/Pressed/Disabled but no Focus.
  • 06 · WCAG 2.4.11Focus visibilityValidates indicator thickness (≥2px) and contrast (≥3:1) against the surface.

Run it locally

  1. Clone and build:
git clone https://github.com/juliocoraspe/figma-a11y-audit.git
cd figma-a11y-audit
npm install
npm run build
  1. In Figma desktop: Plugins → Development → Import plugin from manifest… and pick dist/manifest.json.
  2. For AI alt text, run Ollama with permissive CORS (the plugin iframe has a null origin):
OLLAMA_ORIGINS="*" ollama serve
ollama pull llama3.2-vision
